Design Twitter Feed

A social media timeline like Twitter/X. Users scroll a personalized feed of posts from people they follow, create posts, and like posts. Has ~500M DAU with reads massively dominating writes. ^tIdcHfS1

Functional Requirements
- Scroll home feed (timeline)
- Create a post (tweet)
- Like a post ^sxY5VDcj

Non-Functional Requirements
- High availability (AP) for feed reads
- Read >> Write (10,000:1+)
- Low latency feed loading
- Fan-out at scale: celebrity with 10M followers posts → 10M feeds ^ubWjFthP

Out of Scope
- Retweets / reposts
- Comments / replies
- Direct messages
- Search / trending
- Ranking algorithm ^FE9pDgM2

Deep Dive Optimizations
1. Fan-Out On Write — pre-compute feeds into Redis on post creation, sub-ms feed reads
2. Hybrid Fan-Out — celebrities use fan-out on read to avoid 10M write amplification
3. Feed Cache (Redis) — sorted sets per user, 800 posts, ~6.4 TB cluster
4. Post Cache (Redis) — hydrate postIds to full objects without DB hits
5. Async Like Counter — Redis INCR + periodic flush, avoids hot-row contention
6. Kafka Fan-Out Pipeline — decouples post creation from feed distribution ^R3njMYSx
