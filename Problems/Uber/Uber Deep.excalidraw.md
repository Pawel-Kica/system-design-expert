Deep Dive Optimizations
1. Redis Geospatial (GEOADD/GEOSEARCH) — replaces Postgres for high-frequency location writes and spatial queries
2. Redis Distributed Lock (NX + TTL) — prevents double-assignment of rides with exact timeout
3. WebSocket + Redis Pub/Sub — real-time ride tracking without polling
4. City-based partitioning — reduces working set per query, enables regional deployment
5. Routing API cache — avoids excessive Google Maps API calls for ETA ^YqYCngSS
